# Информационная система учёта автозапчастей

Веб-приложение для управления складом автозапчастей: учёт остатков, приходов, расходов, привязка к маркам автомобилей.

---

## Стек технологий

| Слой | Технология |
|------|-----------|
| Backend | Go 1.25, chi, pgx |
| Frontend | React, Vite, Ant Design |
| База данных | PostgreSQL 16 |
| Инфраструктура | Docker, Docker Compose |

---

## Схема базы данных

### categories — категории запчастей
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL PK | Идентификатор |
| name | VARCHAR | Название (уникальное) |

### brands — марки автомобилей
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL PK | Идентификатор |
| name | VARCHAR | Название (уникальное) |

### parts — справочник запчастей
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL PK | Идентификатор |
| category_id | FK → categories | Категория |
| name | VARCHAR | Название |
| article | VARCHAR | Артикул (уникальный) |
| description | TEXT | Описание |
| created_at | TIMESTAMP | Дата добавления |

### part_brands — связь запчасть ↔ марка (многие ко многим)
| Поле | Тип | Описание |
|------|-----|----------|
| part_id | FK → parts | Запчасть |
| brand_id | FK → brands | Марка |

### stock — текущие остатки
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL PK | Идентификатор |
| part_id | FK → parts | Запчасть (уникальная) |
| quantity | INTEGER | Количество на складе |

### income — приходы
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL PK | Идентификатор |
| part_id | FK → parts | Запчасть |
| quantity | INTEGER | Количество |
| date | DATE | Дата прихода |
| comment | TEXT | Комментарий |

### outcome — расходы
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL PK | Идентификатор |
| part_id | FK → parts | Запчасть |
| quantity | INTEGER | Количество |
| date | DATE | Дата расхода |
| comment | TEXT | Комментарий |

### Связи между таблицами

- `categories` → `parts` — одна категория содержит много запчастей
- `brands` → `part_brands` ← `parts` — запчасть подходит ко многим маркам, марка имеет много запчастей
- `parts` → `stock` — у каждой запчасти одна запись об остатке
- `parts` → `income` — у запчасти много записей о поступлениях
- `parts` → `outcome` — у запчасти много записей о списаниях

### Таблицы

| Таблица | Назначение |
|---------|-----------|
| `categories` | Категории запчастей (Двигатель, Тормоза...) |
| `brands` | Марки автомобилей (Toyota, BMW...) |
| `parts` | Справочник запчастей |
| `part_brands` | Связь запчасть ↔ марка (многие ко многим) |
| `stock` | Текущий остаток по каждой запчасти |
| `income` | История поступлений на склад |
| `outcome` | История списаний со склада |

Приход и расход выполняются в **транзакции**: запись в историю + обновление остатка. При расходе проверяется достаточность остатка.

## API

| Метод | URL | Описание |
|-------|-----|----------|
| GET/POST | `/api/categories` | Список / создать категорию |
| PUT/DELETE | `/api/categories/{id}` | Обновить / удалить категорию |
| GET/POST | `/api/brands` | Список / создать марку |
| PUT/DELETE | `/api/brands/{id}` | Обновить / удалить марку |
| GET/POST | `/api/parts` | Список / создать запчасть |
| GET/PUT/DELETE | `/api/parts/{id}` | Получить / обновить / удалить запчасть |
| GET | `/api/stock` | Текущие остатки |
| GET/POST | `/api/income` | История / добавить приход |
| GET/POST | `/api/outcome` | История / добавить расход |

## Запуск

### Требования
- Docker Desktop ([docker.com](https://www.docker.com/products/docker-desktop/))

### Команды

```bash
# Первый запуск (сборка образов)
docker compose up --build

# Последующие запуски
docker compose up

# Остановить
docker compose down

# Остановить и удалить данные БД
docker compose down -v
```

После запуска открыть в браузере: **http://localhost:8080**
